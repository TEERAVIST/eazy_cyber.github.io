10 commands you need to know to be a beginner

1. kextstat

“kextstat” command in a simple way. Imagine you have a Mac computer, and you want to see what special pieces of software are loaded into your system to help it run. These pieces of software are called kernel extensions, or “kexts” for short.

Think of your Mac as a giant Lego set. The basic Legos are like the operating system, but sometimes you need special pieces to do certain things, like connecting to a printer or using specific hardware. These special pieces are the kexts.

The kextstat command is like a tool that shows you a list of all the special Lego pieces (kexts) currently loaded into your Mac. It helps you see what’s running in the background to make sure everything is working properly.

When you run this command, you’ll see a list of kexts, which might look a bit complicated. Each line in the list shows details about a kext, like its ID, version, and where it came from.

Here’s an example of what part of the output might look like:

Many of the entries in this list say they are loaded at address zero. That just means they are part of the kernel proper and aren’t really device drivers—i.e., they cannot be unloaded. The first actual driver is number 10.

  • Index: This is just a number to keep track of each kext.
  • Refs: This shows how many times the kext is being used.
  • Address: This is where the kext is loaded in memory.
  • Size: This is how big the kext is.
  • Wired: This tells how much memory the kext is using that can’t be swapped out.
  • Name (Version): This is the name of the kext and its version number.
  • Linked Against: This shows other kexts that depend on this one.

2. sysctl

you know how computers have rules, right? Like, “don’t talk to strangers” or “don’t open suspicious emails”? Well, sysctl is like a super cool way to change those rules on a computer, but only for the operating system (OS) itself!

Imagine the OS is like a big building with lots of rooms. Each room has its own set of rules, like “how many people can fit in here” or “what time the lights turn off”. Sysctl is like a special key that lets you go into those rooms and change the rules, but only if you have permission, of course!

Sysctl stands for “system control”, and it’s a way to tweak the OS’s behavior without having to rewrite the whole code. It’s like adjusting the settings on your favorite video game to make it more fun or challenging.

Here are some cool things you can do with sysctl:

  1. Network tweaks: You can change how the computer talks to other computers on the internet. It’s like adjusting the volume on your phone’s speaker!
  2. Security settings: You can make the OS more secure by changing rules about who can access what. It’s like setting up a strong password for your secret diary!
  3. Performance boosts: You can make the computer run faster by adjusting how it uses its resources. It’s like giving your computer a superpower energy drink!

To use sysctl, you need to know some special commands, like sysctl -a to see all the current rules, or sysctl -w to change a specific rule. It’s like using a special decoder ring to unlock the secrets of the OS!

So, that’s sysctl in a nutshell! It’s a powerful tool that helps you customize the OS to your needs, making it more secure, faster, and more fun to use.

Here’s an example of what part of the output might look like:
